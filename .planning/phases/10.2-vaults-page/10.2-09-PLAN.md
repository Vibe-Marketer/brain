Update youtube-import edge function to accept and use vault_id parameter:

```typescript
interface YouTubeImportRequest {
  videoUrl: string;

}
```

After creating the recording in the recordings table, if vault_id provided:
1. Validate vault membership — query vault_memberships for user + vault
2. If no membership: return 403 "Access denied to vault"
3. Create vault_entry: insert into vault_entries with vault_id, recording_id

Backwards compatibility: If vault_id not provided, recording still goes to personal vault (existing behavior via trigger or default).

If the sync functions don't insert recordings directly (e.g., they delegate to another function), follow the chain.
