# Comparison Page Creation Workflow

<objective>
Create a complete "[Your Product] vs [Competitor]" or "[Competitor] Alternative" page optimized for bottom-funnel conversions. These pages capture buyers actively comparing options.
</objective>

<inputs_required>
- Your product name
- Competitor name(s)
- Your key differentiators
- Pricing info (yours and competitor if public)
- Target audience
</inputs_required>

<process>

## Phase 1: Research Competitor

1. Visit competitor's website
2. Document:
   - Core features
   - Pricing tiers
   - Target audience
   - Unique selling points
   - Known weaknesses (from reviews, Reddit, G2)

3. Find competitor reviews on G2/Capterra
   - Note common complaints
   - Note praised features
   - Extract specific quotes if useful

## Phase 3: Write the Content

**Tone Guidelines:**
- Professional and balanced (not salesy)
- Acknowledge competitor strengths (builds trust)
- Be specific with features and pricing
- Use data when available
- Let the comparison speak for itself

**Entity Optimization:**
- Use full product names consistently
- Include specific pricing numbers
- Mention specific features by name
- Reference real customer segments

**Snippet Optimization:**
- Comparison table at top (table snippet)
- "TL;DR" section for paragraph snippet
- FAQ section for PAA capture

## Phase 4: Add Conversion Elements

- CTA after quick comparison table
- CTA mid-page after your product section
- CTA at end
- "Start free trial" or "See demo" (low commitment)

## Phase 5: Schema & Meta

**Meta Title:**
`[Your Product] vs [Competitor]: [Year] Comparison | [Your Brand]`

**Meta Description:**
`Comparing [Your Product] and [Competitor]? See our detailed breakdown of features, pricing, and use cases to find the right fit for [target audience].`

**Schema:**
- Article schema
- FAQPage schema for FAQ section
- Product schema (optional, for your product)

</process>

<success_criteria>
- Page is 2000-3500 words
- Comparison table included
- Fair to competitor (not attack piece)
- Clear recommendation for different use cases
- Multiple CTAs placed naturally
- FAQ section with 5+ questions
- Schema markup specified
- Saved as .md file ready to publish
</success_criteria>
